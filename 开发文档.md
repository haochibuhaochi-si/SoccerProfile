# 任务：开发足球风格与竞技水准测试 H5 独立站

## 1. 项目概述
开发一个移动端优先（Mobile First）的 H5 网站。用户回答 35 道足球相关问题，提交答案后，系统计算用户的足球风格（Archetype）、竞技水准（PlayLevel）和战术素养（Literacy）。用户需支付（或观看广告）后方可查看详细报告。

## 2. 技术栈要求
*可使用腾讯云开发cloudbase相关组件完成H5独立站相关开发。   
*   **数据存储**：无需复杂数据库，初期可使用 JSON 文件存储文案包（`copy_pack.json`）。
*   **部署**：静态文件托管在腾讯云cloudbase上

## 3. 页面流程与功能

### Step 1: 欢迎页 (Welcome)
*   展示 Logo 和 Slogan。
*   按钮：“开始测试”。

### Step 2: 答题页 (Quiz)
*   **进度条**：显示当前题号/总题号 (35)。
*   **题目渲染**：
    *   Part 0 (Q0): 多选框 (Checkbox)，最多选2项。
    *   Part 1 (P1-P19): 单选题 (Radio)，风格测试。
    *   Part 2 (S1-S10): 单选题，强度测试。
    *   Part 3 (K1-K5): 单选题，素养测试。
*   **交互**：点击选项后自动跳转到下一题（或手动点击下一步）。

### Step 3: 加载页 (Loading)
*   用户提交答案后，显示加载动画。
*   文案：“正在分析你的球风...”、“正在匹配 Archetype...”。

### Step 4: 付费墙 (Paywall)
*   展示报告预览图（模糊处理）或亮点摘要。
*   展示价格/解锁方式（通过连接微信平台JSAPI完成微信支付或可考虑观看视频广告）。
*   按钮：“解锁我的报告”。

### Step 5: 报告页 (Report)
*   **Header**: 展示 Archetype 名称 (如 "创意型全能中场") + 强度标签 (如 "标准业余级") + 素养标签 (如 "🧠 战术大师")。
*   **Radar Chart**: 使用 Canvas 或 SVG 绘制风格雷达图（6-8个维度）。
*   **Core Analysis**: 展示深度解析、优势、劣势。
*   **Competitive Analysis**: 展示强度分析。
*   **Training Advice**: 展示训练建议。
*   **Style Reference**: 展示风格参照系（注意使用方案 A，避免侵权）。
*   **Footer**: 分享按钮、重新开始按钮。

## 4. 核心算法逻辑 (Backend)
后端必须实现以下逻辑（参考 `calculation_rules.md`）：

1.  **Input**: 接收 35 个问题的答案数组。
2.  **Style Vector**: 根据标签（如 F2-Engagement）计算每个轴的得分并归一化。
3.  **Archetype Matching**:
    *   根据 Q0 的位置过滤候选集。
    *   计算与 13 个原型向量的余弦相似度，找出 Top 2。
    *   若副型相似度 ≥ 主型 × 0.85，标记为混合亚型 (Hybrid)。
4.  **PlayLevel Calculation**:
    *   Env Tier (S1) 定基准；S2 频率、S3 队友、S5 可靠性、S8 场地做修正。
    *   Exec (S4, S6, S7) 做执行修正。
    *   Literacy (K1-K5) 做素养加成。
5.  **Output**: 返回一个 JSON 对象，包含：
    *   `archetype_id`, `archetype_title`, `is_hybrid`, `hybrid_desc`.
    *   `playlevel_score`, `intensity_band`.
    *   `literacy_band`.
    *   `radar_data` (数组).
    *   `copy_data` (从 JSON 文件中读取的文案).

## 5. 文案与数据
*   我已经为你准备好了 `question.md` (35题) 和 `copy_pack.json` (所有文案)。
*   后端程序需要读取 `copy_pack.json` 来填充报告内容。
*   **重要**：报告中的“风格参照系”必须使用 `style_reference` 字段，并在顶部显示免责声明。

## 6. 设计风格
*   **色调**：深色模式（Dark Mode）为主，搭配高饱和度的强调色（如电光蓝、警示橙）。
*   **字体**：使用无衬线字体，清晰易读。
*   **动效**：平滑的页面切换，数字滚动效果（如 PlayLevel 5.8）。

## 7. 注意事项
*   确保移动端键盘不会遮挡输入框（虽然本题无输入，但需考虑兼容性）。
*   处理好网络延迟，所有请求需有 Loading 状态。
*   代码需注释清晰，方便后续维护。
